GOMS Model

GOMS stands for Goals, Operators, Methods, and Selection rules. It predicts how users perform tasks by breaking down interactions.

  • G – Goals: What the user wants to achieve. Example: “Print a document.”
  • O – Operators: Basic actions the user performs to achieve a goal. Example: Clicking a button, typing a character, moving the mouse.
  • M – Methods: Sequences of operators that accomplish a goal. Example: To print a document: File → Print → Select Printer → Click Print.
  • S – Selection rules: Guidelines for choosing among multiple methods to achieve the same goal. Example: “If the printer is available, use Method A; else use Method B.”

    • KLM (Keystroke-Level Model)

    KLM estimates task completion time by summing keystroke and mouse actions for skilled users.

    Norman’s Gulf of Execution & Evaluation

    Users form goals and try to execute them in the system. Feedback is crucial for closing the gulf of evaluation.
